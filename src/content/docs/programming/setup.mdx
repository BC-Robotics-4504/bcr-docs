## Prerequisites

- Windows 11 laptop
- Internet access
- Admin privileges
- Basic command line knowledge

## 1. Install IntelliJ IDEA

<Aside type="note" title="Why IntelliJ?">
  IntelliJ IDEA is a powerful IDE for JVM and cross-language development with first-class Gradle support,
  robust Git integration, and excellent debugging. Our team prefers IntelliJ for robot code development.
</Aside>

1. Download IntelliJ IDEA:
   - https://www.jetbrains.com/idea/download/
   - Choose **Community** (free) or **Ultimate** (if you have a license) ‚Üí **Windows x64**.
2. Run the installer and follow the prompts. Accept defaults unless you have a personal preference.
3. After installation, open IntelliJ and install or enable these recommended plugins:
   - Docker (for interacting with containers)
   - Gradle (usually built-in)


4. Configure the JDK / Project SDK:
   - Install the team‚Äôs agreed JDK version (ask the team if unsure).
   - In IntelliJ: File ‚Üí Project Structure ‚Üí SDKs ‚Üí Add the installed JDK.
   - When opening a Gradle-based robot project, IntelliJ will prompt to use an SDK ‚Äî select the team JDK.

5. Clone and open the repo:
   - Use IntelliJ‚Äôs VCS menu: VCS ‚Üí Get from Version Control ‚Üí paste (replace OUR_REPO with the repo name):
     https://github.com/BC-Robotics-4504/OUR_REPO
   - Or clone on the command line and open the folder in IntelliJ:
   ```bash
   git clone https://github.com/BC-Robotics-4504/bcr-docs
   cd bcr-docs
   ```
   - Open the project in IntelliJ (File ‚Üí Open)

<Aside type="tip">
  If you prefer the exact Dev Container workflow (automatic environment provisioning), VS Code's Dev Containers integration is more seamless.
  IntelliJ can still work with Docker/WSL via the Docker plugin and remote toolchains, but that requires extra configuration.
</Aside>

## 3. Install WPILib

WPILib is the core library and tooling for FRC robot programming.

1. Go to the official releases:
   üëâ [WPILib GitHub Releases](https://github.com/wpilibsuite/allwpilib/releases)
2. Download the latest **WPILib Installer for Windows**.
3. Run the installer. It will:
   - Install VS Code FRC extensions
   - Install the RoboRIO imaging tool
   - Set up Gradle for robot builds

<Aside type="caution" title="Important">
  Always use the **team‚Äôs agreed WPILib version** during build season. Different versions can cause
  build errors.
</Aside>

## 4. Install OpenJDK 17

OpenJDK 17 (LTS) is the recommended Java runtime for our Gradle and robot builds.

1. Download Eclipse Temurin (OpenJDK 17) from Adoptium:
  üëâ https://adoptium.net/temurin/releases/?version=17
  - Choose the Windows x64 MSI installer for an easy install.

2. Run the MSI installer and accept defaults.
  - If the installer offers to set JAVA_HOME, enable that option.

3. If JAVA_HOME wasn't set automatically, add it manually:
  - Settings ‚Üí System ‚Üí About ‚Üí Advanced system settings ‚Üí Environment Variables
  - Under System variables: New
    - Variable name: JAVA_HOME
    - Variable value: C:\Program Files\Eclipse Adoptium\jdk-17.x.x (adjust for the installed folder)
  - Edit the System PATH and add: %JAVA_HOME%\bin

4. Restart any open terminals (or your PC) so environment changes take effect.

5. Verify the installation:

PowerShell:
```sh
javac --version
```

## 5. Install Docker Desktop

Docker lets us run **Dev Containers** for a consistent environment across all machines.

1. Download from [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/).
2. Run the installer, accept defaults.
3. Enable **WSL 2 backend** when prompted.
4. Restart your computer.
5. Verify Docker is running from the system tray.

Check Docker works:

```bash
docker --version
```

---

## 6. Install WSL (Windows Subsystem for Linux)

WSL provides a Linux environment on Windows. This is what Dev Containers use.

1. Open **PowerShell** as Administrator.
2. Run:

```powershell
wsl --install
```

3. This installs **Ubuntu** by default. Restart when prompted.
4. After reboot, set a username/password for your Ubuntu environment.
5. Update Ubuntu packages:

```bash
sudo apt update && sudo apt upgrade -y
```

6. Verify:

```bash
wsl --list --verbose
```
